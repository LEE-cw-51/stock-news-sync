## 세션 시작 브리핑

이 스킬은 **04번(Tech Lead PM)만** 실행한다.

---

### 실행 규칙

1. `docs/status/HANDOFF.md`와 `docs/status/PROGRESS.log`를 **Read 도구로만** 읽는다.
2. Lint, 빌드 테스트, 파일 스캔, git diff 등 **무거운 검증은 절대 실행하지 않는다.**
3. 파일이 없거나 비어 있으면 "이전 세션 기록 없음"으로 처리한다.
4. 브리핑 후 사용자의 다음 지시를 기다린다.

---

### 브리핑 양식

```
☀️ [04 Tech PM] 세션 브리핑
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
완료된 작업:    {HANDOFF.md completed 섹션 요약 1줄, 없으면 "기록 없음"}
미해결 이슈:    {HANDOFF.md blockers 섹션 요약, 없으면 "특이사항 없음"}
오늘 1순위:     {HANDOFF.md next 섹션 또는 PROGRESS.log 기반 가장 중요한 작업 1가지}
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
👉 1순위 작업을 바로 시작할까요, 아니면 다른 명령을 내리시겠습니까?
```

---

### 트리거 문구

사용자가 다음 문구로 대화를 시작하면 이 스킬을 실행한다:
- "출근했어"
- "세션 시작"
- "안녕" (대화의 첫 메시지인 경우)
